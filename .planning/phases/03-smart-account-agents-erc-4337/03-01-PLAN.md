<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EntryPoint → account.validateUserOp | EntryPoint is the only trusted caller of validateUserOp; account must verify msg.sender |
| Owner key → signed UserOp | Only the owner's ECDSA signature authorizes ops; sig recovered from untrusted bytes |
| External caller → execute/executeBatch | Anyone could try to call these directly to drain the account |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | CascadeAccount.execute/executeBatch | mitigate | `onlyEntryPointOrOwner` modifier requires msg.sender == entryPoint || owner |
| T-03-02 | Spoofing | CascadeAccount.validateUserOp | mitigate | `require(msg.sender == address(entryPoint))` at function entry |
| T-03-03 | Tampering | inline ecrecover (signature) | mitigate | malleability guard — reject high-s, v not in {27,28}, and zero-address recover (no OZ in lib/, so guard inline) |
| T-03-04 | Denial of Service | _payPrefund | accept | pay only missingAccountFunds, ignore call success (EntryPoint verifies); account funded minimally — low value target |
| T-03-SC | Tampering | package installs | accept | NO package-manager installs in this phase (interfaces hand-written); lib/ unchanged (forge-std only) — supply-chain surface is zero |
</threat_model>
