    - testFuzz_linear_chain_conserves(depthSeed): build a chain of bound(depthSeed,1,8) skills each routing a fuzzed share into the prior; invoke top; assert summed balances == price.

    - invoke(seed): pick a registered skill, vm.deal the exact price to a chosen actor, prank-invoke; accumulate ghost_totalPaidIn += price.

    - invariant_accrued_equals_paid: sum(balances over creators) + ghost_totalClaimed == ghost_totalPaidIn.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| randomized fuzz input -> register/invoke bounds | Out-of-bound input must be clamped (bound), not silently discarded, or the property goes untested |
| handler ghost accounting -> contract state | Ghost totals are the oracle; if they drift from real calls the invariant is vacuous |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Information disclosure | vacuous invariants (handler reverts on every call) | mitigate | Use bound() not vm.assume() so calls actually execute; assert >=3 invariants reference real balances + ghost totals |
| T-04-05 | Tampering | non-deterministic gas snapshot (fork RPC in snapshot) | mitigate | Snapshot excludes fork tests (--no-match-path); committed offline-reproducible baseline |
| T-04-06 | Denial of service | unbounded fuzz price overflowing vm.deal | accept | Price bounded to a sane range; conservation property unaffected by the cap |
</threat_model>
